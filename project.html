Neighbor Connect - Benefactor View

Role: UI Designer

Goal: to create an accessable interface for the benefactor for a benevolent task service.

Strategy: to identify each user view and it's interaction with the benefactor and spider web out from the beefactor to the other users.

Process

Starting with seperating the interface views into four groups: Neighbor, Benefactor, Local Admin, and Global Admin.

When this project came in it was going to be a quick single page web form. After meeting with the client a few times, we realized their initial project request was not actually the solution they needed for their problem. The project was passed on to a development team, but I was asked to continue to develop the userflow for the benefactor view for use in the completed application, and as a visual guide for completed app.
